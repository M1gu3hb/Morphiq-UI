# Reportes de ejecución

Cada vez que un ejecutor (Claude Code o Codex) termina un prompt/tarea, deja aquí un reporte.

- Copia `TEMPLATE.md` y renómbralo `NNNN-slug.md` (secuencial, 4 dígitos): p. ej. `0002-fix-timeline-drag.md`.
- Llena todas las secciones con la verdad: qué se pidió, qué se hizo, resultado esperado vs. real, bugs y fixes, gate corrido, riesgos.
- El orquestador/auditor lee estos reportes y los compara contra el código real. Reportes inflados o imprecisos se detectan en esa comparación.

Flujo completo del equipo en [`../workflow.md`](../workflow.md).
